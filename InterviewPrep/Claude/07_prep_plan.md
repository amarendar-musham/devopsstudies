# 3-Week Prep Plan + Day-Of Checklist

> Adjust to your timeline. If you have interviews sooner, front-load Week 1 items (behavioral + resume + system design = highest ROI).

---

## WEEK 1 — Foundation (your story + your resume)
**Goal:** never get caught off guard on your own work.
- [ ] Read `01_behavioral_star.md`; rehearse all 8 stories out loud (2 min each).
- [ ] Write the 4 **gap stories** (failure, conflict, influence, leadership) — draft in STAR.
- [ ] Read `02_resume_deep_dive.md`; answer every follow-up aloud. Flag weak bullets → study them.
- [ ] Skim `04_technical_qna.md` once to see what you already know vs need work.
- [ ] Daily: 1 LeetCode Easy to warm up (`05_coding_dsa.md`).

## WEEK 2 — Technical depth
**Goal:** system design fluency + core fundamentals.
- [ ] Read `03_system_design.md`. Do **all 6 prompts**, timed 35 min, out loud + drawing.
- [ ] Drill `04_technical_qna.md` — cover answers, explain each aloud. Re-drill misses.
- [ ] Daily: 1–2 LeetCode (mix Easy/Medium), focus hashmap/string/graph.
- [ ] Practice 2 SRE-style scripting problems (log parse, LRU cache).

## WEEK 3 — Integration + mock
**Goal:** perform under interview conditions.
- [ ] 2–3 full mock rounds (behavioral + design + coding). Record yourself or use a friend.
- [ ] Re-do the design prompts you were weakest on.
- [ ] Finalize `06_questions_to_ask.md` — pick your favorites per round type.
- [ ] Light review only in the last 2 days — don't cram new material.
- [ ] Research each specific company: stack, recent news, the team/role.

---

## PER-INTERVIEW PREP (the day before)
- [ ] Re-read the JD; map your stories/skills to their requirements.
- [ ] Research the company's tech stack + recent engineering blog posts.
- [ ] Pick 2–3 questions to ask (from file 06), tailored to the round.
- [ ] Pick which 3–4 STAR stories fit this role best.
- [ ] Sleep. Don't cram.

## DAY-OF CHECKLIST
- [ ] Test setup: camera, mic, internet, quiet room, water.
- [ ] Resume + notes open (for virtual rounds): your STAR one-liners, questions to ask.
- [ ] Paper + pen for design/coding scratch.
- [ ] Join 5 min early.
- [ ] Deep breath. Think out loud. Clarify before answering. It's a conversation.

## DURING THE INTERVIEW — reminders
- **Behavioral:** STAR structure, "I" vs "we", lead with result if time-boxed.
- **Design:** clarify → scope → high-level → deep-dive → failure/scale → trade-offs.
- **Coding:** restate → clarify → brute-force → optimize → code → test → complexity.
- **Always:** it's fine to pause and think. "Let me think about that for a second" is a strong move.

## AFTER EACH INTERVIEW
- [ ] Jot down questions asked + how you did → feeds next prep.
- [ ] Send a brief thank-you if you have the interviewer's contact (optional in India, but nice).

---

## PRIORITY IF YOU'RE SHORT ON TIME
1. Behavioral stories (01) + resume deep-dive (02) — you WILL be asked these.
2. System design framework + 2–3 prompts (03).
3. Core technical Q&A: Kubernetes, CI/CD, Cloud (04).
4. Coding only if the company does DSA (05).
