Ref: https://kubernetes.io/docs/setup/production-environment/tools/kubeadm/create-cluster-kubeadm/

On all nodes: yum install docker container runtime, Ensure docker service running
On all nodes: yum install kubelet, kubeadm, kubectl on all nodes; enable kubelet service

On all nodes: disable swap (free -m, /etc/fstab, disable swap mount)
On all nodes: disable firewall or open appropriate ports in the firewall

On all nodes: /etc/hosts, setup hostname resolving
##/etc/hosts # { IP Domain alias}
## ssh alias

On control: kubeadm init. (root), note down "kubeadm join" command after initialization.

===========================
client configuration file creation:(regular user)
 - mkdir -p $HOME/.kube
 - cp -l /etc/kubernetes/admin.conf $HOME/.kube/config
 - chown $(id -u):$(id -g) $HOME/.kube/config
 - kubectl cluster-info
 
 Alternative:
 export KUBECONFIG=/etc/kubernetes/admin.conf

===========================
Installing a Pod network add-on (Only one pod n/w per cluster)
kubectl apply -f <>.yaml ##  Ex: flannel/calico/Weave 

Ensure namespace/kube-system - pod/{coredns,flannel} is running. If it is up and running, you can start joining nodes. 
 
=========================== 
--- Join nodes in cluster
kubeadm join --token<> IP:6443 --discovery-token-ca-cert-hash <> ## if no token -> kubeadm token list/create/
